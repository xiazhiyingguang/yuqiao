# 语桥“医学参考型沟通画像”首次配置方案 V1.0

## 1. 产品定位

语桥首次配置不是失语症诊断，也不输出轻度、中度、重度结论。

系统以失语症临床评估中的核心语言维度为参考，结合用户自述、轻量任务体验和日常沟通目标，生成“沟通能力画像”，自动调整候选数量、界面密度、图片比例、语音辅助、首页入口与家属协作方式。

输出原则：

* 不诊断疾病。
* 不判断失语类型。
* 不替代言语治疗师。
* 不展示“失败”“答错”等负面表达。
* 所有配置都允许用户、家属或治疗师随时修改。
* 用户手动选择优先级高于系统推荐。

---

## 2. 首次使用总流程

总时长控制在 5—8 分钟，可随时跳过、暂停或由家属协助完成。

流程：

1. 安全提示
2. 选择使用身份
3. 已有医学评估信息导入
4. 使用条件检查
5. 日常沟通目标选择
6. 六项轻量沟通任务
7. 界面密度试用
8. 生成沟通能力画像
9. 用户确认或调整
10. 进入个性化首页

---

## 3. 第 0 屏：安全提示

标题：

“先确认一下”

内容使用大字、图标和语音朗读：

“如果你是刚刚突然出现说不清、听不懂、嘴歪、一侧手脚无力或明显加重，请先联系家人、医生或急救服务。语桥不能用于紧急诊断。”

按钮：

* 我现在安全，继续
* 联系家人
* 我以后再设置

设计要求：

* 不强制用户填写病史。
* 不要求用户回答医学问题。
* 仅做安全提醒，不保存该页面选择。

---

## 4. 第 1 屏：选择当前使用方式

标题：

“谁在帮忙设置？”

四个大按钮：

* 我自己设置
* 家人帮我设置
* 治疗师帮我设置
* 先用默认简单模式

不同入口：

| 使用方式   | 后续界面                   |
| ------ | ---------------------- |
| 用户本人   | 语音朗读、短句、图片优先           |
| 家属     | 增加家庭场景、常用人物、常用物品配置     |
| 治疗师    | 显示专业评估导入入口             |
| 默认简单模式 | 直接进入 2 候选、大图片、固定高频表达首页 |

---

## 5. 第 2 屏：已有评估信息导入

标题：

“你以前做过语言评估吗？”

选项：

* 医院或治疗师已经评估过
* 我不确定
* 还没有做过
* 现在先跳过

若选择“已经评估过”：

显示专业人员模式入口：

* 量表类型：WAB / BDAE / ABC / CRRCAE / CQAB / 其他
* 评估日期
* 主要困难：命名 / 听理解 / 组织句子 / 阅读 / 书写 / 其他
* 是否允许用于优化语桥界面
* 是否需要家属查看结果

重要规则：

* 不要求患者自己输入复杂医学分数。
* 不让用户自行理解量表分数。
* 不把导入分数转化成“轻度/中度/重度”标签。
* 显示“本信息来自专业评估，语桥仅据此调整辅助方式”。

---

## 6. 第 3 屏：使用条件检查

标题：

“怎样使用会更舒服？”

每题只显示一个问题和大按钮。

### 6.1 听觉条件

“你现在听手机声音舒服吗？”

* 很舒服
* 有一点听不清
* 希望看文字和图片
* 希望家人帮忙

系统作用：

* 听觉困难时，增加文字、图片和重复播放按钮。
* 不把听觉困难误判成语言理解障碍。

### 6.2 视觉条件

“看手机上的字舒服吗？”

* 舒服
* 希望字更大
* 希望图片更多
* 看起来容易累

系统作用：

* 调整字体大小、图片比例、背景对比度和候选数量。

### 6.3 手部操作条件

“点手机屏幕方便吗？”

* 方便
* 有一点困难
* 希望按钮更大
* 希望少一些选项

系统作用：

* 调整点击区域、候选数量和底部导航复杂度。

### 6.4 疲劳状态

“今天的状态怎么样？”

* 精神不错
* 有一点累
* 很累，想简单一点

系统作用：

* 很累时自动推荐“低负担模式”：2 个候选、少动画、固定高频表达、短句优先。

---

## 7. 第 4 屏：选择最常见的沟通困难

标题：

“你最希望语桥帮什么？”

允许多选，每项配图片和语音。

* 我知道意思，但想不起词
* 我能说一些词，但不容易说成一句话
* 别人说快了，我不容易听懂
* 看很多字会累
* 我说话不清楚，别人不容易明白
* 我想表达日常需要
* 我想和家人聊天
* 我想在医院表达需求
* 我想外出时更方便沟通

系统映射：

| 用户选择 | 优先功能                      |
| ---- | ------------------------- |
| 想不起词 | 拍照找词、人物与物品词库、个人物品记忆       |
| 说不成句 | “我想—我需要—请帮我”句式卡片          |
| 听不懂  | “请说慢一点”“请再说一次”“用图片告诉我”快捷语 |
| 看字累  | 图片优先、短句、慢速朗读              |
| 说不清楚 | TTS 播报、图文表达、减少语音输入依赖      |
| 医院需求 | 疼痛、如厕、喝水、呼叫护士、用药、联系人快捷表达  |

---

## 8. 第 5 屏：六项轻量沟通任务

原则：

* 不叫“测试”。
* 不显示分数。
* 不显示对错。
* 每题可跳过。
* 每题都支持“再听一次”“我想休息”“家人帮我”。
* 用户完成的是界面体验任务，不是医学考试。

### 任务 A：词语理解体验

页面提示：

“听一听，选出你听到的东西。”

示例：

手机播放“杯子”，界面显示“杯子”和“遥控器”两张真实图片。

记录：

* 是否一次完成
* 是否点击重复播放
* 是否需要家属帮助
* 是否主动跳过

对应能力维度：

词语听理解。

系统用途：

决定是否增加图片、重复朗读和短句解释。

---

### 任务 B：句子理解体验

页面提示：

“听一听，选出最接近的一张图。”

示例：

“把杯子放在桌上。”

界面显示两张不同动作图片。

记录：

* 是否理解简单动作关系
* 是否需要重复
* 是否适合使用较长句子

对应能力维度：

句子听理解。

系统用途：

决定对话理解模块使用“原句 + 短句解释”还是“原句 + 图片关键词”。

---

### 任务 C：找词体验

页面提示：

“看到这个，你会怎么说？”

展示真实生活照片，例如水杯、药盒、门、遥控器。

按钮：

* 我能说出来
* 我知道是什么，但一下想不起名字
* 我想用图片表达
* 我不确定

不使用自动语音评分。

对应能力维度：

命名与找词。

系统用途：

“我知道但说不出”被频繁选择时，首页优先展示“拍照找词”“我的物品”“人物词库”。

---

### 任务 D：组织一句话体验

页面提示：

“你想喝水时，哪种方式最容易表达？”

界面提供三种表达方式：

* “我想喝水”
* “我想 + 喝水”
* 图片：我 + 喝水

用户选择最容易的方式。

对应能力维度：

句子组织和表达支持需求。

系统用途：

决定默认使用完整固定句、动态槽位句式，还是图文组合表达。

---

### 任务 E：阅读理解体验

页面提示：

“看一看，选出和文字一样的图片。”

示例：

文字：“请帮我开门。”

界面展示开门、喝水、坐下三张图片。

按钮：

* 字太小
* 想听一遍
* 我想看图片
* 跳过

对应能力维度：

阅读理解。

系统用途：

决定文字字号、文字与图片比例、TTS 默认开关。

---

### 任务 F：表达密度体验

页面提示：

“哪一种最容易选？”

同一目标词汇使用三种布局展示：

* 2 个大候选
* 4 个标准候选
* 6 个较多候选

每一页都让用户找同一个目标，例如“水”。

记录：

* 用户主观偏好
* 首次点击是否准确
* 是否点击返回或犹豫
* 是否选择“我想要更少”
* 是否选择“我想要更多”

对应能力维度：

视觉搜索、注意力、操作便利性和认知负担。

系统用途：

决定默认候选数量。

---

## 9. 沟通支持指数算法

系统不计算“失语症严重程度”，只计算“支持需求”。

每个维度使用三级支持指数：

| 分值 | 含义      | 用户可见文案    |
| -- | ------- | --------- |
| 0  | 当前可独立完成 | 目前比较顺手    |
| 1  | 需要少量支持  | 加一点帮助会更轻松 |
| 2  | 需要较多支持  | 建议优先提供辅助  |

核心维度：

* 找词支持
* 句子组织支持
* 听理解支持
* 阅读支持
* 语音输出支持
* 视觉支持
* 手部操作支持
* 疲劳支持

计算原则：

1. 用户主动选择优先级最高。
2. 任务体验只用于辅助推荐。
3. 听觉、视觉、手部操作问题不能直接当成语言障碍。
4. 任一任务跳过时，不输出负面结论。
5. 家属或治疗师可修改任何结果。
6. 所有变化都保留“为什么这样设置”的解释。

建议逻辑：

```text
最终支持需求 = max（用户主观选择，任务体验信号）

若听觉或视觉条件异常：
不提高语言障碍判断，只提高界面辅助等级。

若用户选择“今天很累”：
临时开启低负担模式，不修改长期能力画像。
```

---

## 10. 候选数量规则

候选数量不应该由医学量表分数直接决定，而应由“用户是否容易看、容易选、容易理解”决定。

默认推荐：

| 条件                 | 默认候选数量 |
| ------------------ | ------ |
| 听理解、视觉、疲劳或操作支持较高   | 2 个    |
| 需要适量选择，但能处理常用表达    | 4 个    |
| 用户明确希望更多表达空间，且体验顺畅 | 6 个    |

重要规则：

* 用户可以随时切换 2 / 4 / 6。
* 首次配置中系统只能“推荐”，不能强制。
* 不根据某一次点击失败自动减少候选。
* 若连续多次出现犹豫、误触或“都不是”，只弹出建议：“要不要试试更少一些选项？”
* 候选数量可随疲劳状态临时改变。

首页快捷入口：

“选项数量：少 / 标准 / 多”

对应：

* 少：2 个
* 标准：4 个
* 多：6 个

这样用户不需要理解数字含义。

## 11. 沟通画像结果页

标题：

“语桥已经为你准备好了”

示例结果：

### 当前表达模式：图片辅助表达

你更容易通过图片和短句完成表达。

已开启：

* 首页优先显示拍照找词
* 每页默认显示 2 个候选
* 人物、物品和常用句优先
* 文字自动放大
* 每句话都可以慢速朗读
* “我卡住了”放在首页
* 自动卡顿提醒暂时关闭

为什么这样设置：

* 你选择了“图片更容易看懂”
* 你觉得 2 个选项更容易选择
* 你常遇到“知道意思但想不起词”

按钮：

* 这样很好
* 我要更简单
* 我想要更多选项
* 让家人帮我调整
* 以后再改

## 12. 三种默认沟通模式

### 模式一：简洁表达模式

适用：

* 今天疲劳明显
* 看字困难
* 手部操作不方便
* 希望快速表达基础需求

默认设置：

* 2 个大候选
* 真实图片优先
* 首页保留高频表达
* 关闭自动卡顿提醒
* 固定句优先
* 大字号与高对比度

首页模块：

* 我需要帮助
* 喝水 / 吃饭 / 如厕
* 疼 / 累 / 不舒服
* 找家人
* 拍照找词
* 我卡住了

---

### 模式二：辅助表达模式

适用：

* 想不起词
* 句子不容易组织
* 能完成简单点击和选择

默认设置：

* 4 个候选
* 动态槽位补词
* “我想—我需要—请帮我”句式
* 个人物品和地点词汇
* 手动“我卡住了”
* 候选确认后播报

---

### 模式三：丰富表达模式

适用：

* 理解与阅读能力相对保留
* 希望表达更多细节
* 用户明确喜欢更多候选

默认设置：

* 6 个候选
* 对话上下文补词
* 常用表达排序
* 人物、地点、活动词库
* 可开启轻提醒型卡顿辅助
* 支持拍照、语音与文字混合输入

---

## 13. 与现有语桥功能的接入方案

### 13.1 首页接入

现有首页保留“补词、拍照、对话、词库”能力，但按画像动态排序。

| 支持重点    | 首页第一入口 |
| ------- | ------ |
| 找词支持高   | 拍照找词   |
| 句子组织支持高 | 我想说    |
| 听理解支持高  | 听懂对方   |
| 语音输出支持高 | 点击播报   |
| 疲劳支持高   | 立即表达   |
| 家属协作需求高 | 家人帮我设置 |

---

### 13.2 动态补词接入

在现有动态槽位状态机中增加：

```dart
candidateCount: 2 | 4 | 6
preferredInputMode: photo | icon | text | mixed
sentenceSupportLevel: 0 | 1 | 2
wordFindingSupportLevel: 0 | 1 | 2
```

规则：

* 找词支持高：优先展示照片、个人物品、地点词。
* 句子支持高：优先展示完整句式和固定表达。
* 阅读支持高：每个候选增加图标与朗读按钮。
* 疲劳支持高：减少槽位层级，优先进入常用句。
* 候选生成时请求数量比界面展示多 2 个，用于筛掉重复、无关或不安全候选。

---

### 13.3 对话模式接入

自动卡顿提醒改为三档：

* 关闭：只显示“我卡住了”按钮。
* 轻提醒：疑似卡顿时显示静默小卡片，不震动、不打断。
* 自动辅助：满足多信号条件后轻震并展示候选。

自动辅助仅在以下条件满足时开放：

* 用户或家属主动开启。
* 用户理解“系统可能猜错”。
* 每次候选都需要确认。
* 可随时关闭。
* 系统连续被拒绝时自动降低触发频率。

---

### 13.4 家属模式接入

家属模式通过 PIN 保护。

功能：

* 添加亲友照片、昵称、称呼。
* 添加药盒、杯子、遥控器等常用物品。
* 配置“早晨、吃饭、医院、睡前、外出”场景。
* 编辑紧急表达。
* 调整候选数量、字号、图片比例。
* 查看“沟通提示卡”。

沟通提示卡内容：

* 一次只说一句。
* 说慢一点。
* 给对方足够时间。
* 不替对方接话。
* 不确定时用“是 / 否”问题确认。
* 可以画图、指图片、写关键词。
* 听懂后重复确认一次。

---

## 14. 专业人员协作模式

专业人员模式不在普通用户首页展示。

入口：

“治疗师设置”

需要：

* PIN 或机构账号
* 用户明确授权
* 本地加密保存
* 可随时删除

可录入字段：

```text
assessmentSource
assessmentTool
assessmentDate
assessorName
languageBackground
namingStatus
auditoryComprehensionStatus
repetitionStatus
readingStatus
writingStatus
motorSpeechStatus
functionalCommunicationGoal
clinicalNotes
```

系统输出：

* 专业人员原始记录
* 语桥界面推荐
* 用户实际使用偏好
* 家属协作建议

系统不得输出：

* 新的医学诊断
* 疾病进展结论
* 药物建议
* 康复疗效承诺
* 自动失语类型判定

## 15. 数据结构建议

```dart
class CommunicationProfile {
  final String profileId;
  final DateTime createdAt;
  final String source; // self, caregiver, clinician
  final bool hasClinicalAssessment;
  final String? clinicalTool;
  final DateTime? clinicalAssessmentDate;

  final int wordFindingSupport;
  final int sentenceBuildingSupport;
  final int auditoryComprehensionSupport;
  final int readingSupport;
  final int motorSpeechSupport;
  final int visualSupport;
  final int motorOperationSupport;
  final int fatigueSupport;

  final int preferredCandidateCount; // 2, 4, 6
  final String preferredRepresentation; // photo, icon, text, mixed
  final String stuckAssistMode; // off, gentle, automatic
  final List<String> priorityScenes;
  final List<String> priorityGoals;

  final DateTime lastReviewedAt;
  final bool userConfirmed;
}
```

每个设置增加来源字段：

```dart
enum SettingSource {
  userChoice,
  taskExperience,
  caregiverChoice,
  clinicianInput,
  systemSuggestion,
}
```

这样每项设置都可以解释：

“当前默认显示 2 个候选，因为：用户主动选择 + 界面体验更顺畅。”

## 16. 复评与动态调整

复评不是再次诊断，而是重新确认当前使用体验。

建议频率：

* 发病或康复早期：每 14 天提醒一次。
* 稳定期：每 30 天提醒一次。
* 用户感觉明显变化：随时手动重做。
* 每天只做“疲劳状态”轻量调整，不修改长期画像。

轻量复评内容：

* 今天选项多不多？
* 图片够不够？
* 最近最常卡在哪里？
* 有没有新的常用人物、物品、地点？
* 自动提醒会不会打扰？
* 想不想试试更多或更少选项？

## 17. 开发优先级

### 第一阶段：比赛可展示版本

必须完成：

1. 沟通画像首次配置流程。
2. 2 / 4 / 6 候选数量动态切换。
3. 图片 / 图标 / 文字显示偏好。
4. “今天很累”低负担模式。
5. 首页模块动态排序。
6. 手动“我卡住了”入口。
7. 结果页“为什么这样设置”解释卡。
8. 家属模式基础入口。

### 第二阶段：真实可用版本

继续完成：

1. 专业人员协作模式。
2. 医学评估记录导入。
3. 个人物品和人物照片词库。
4. 沟通伙伴提示卡。
5. 画像复评机制。
6. 本地加密存储。
7. 数据删除与导出功能。

### 第三阶段：机构合作版本

继续完成：

1. 治疗师后台。
2. 标准化评估协作流程。
3. 用户长期沟通趋势记录。
4. 医院与康复机构试点。
5. 多方言与地区词汇适配。
6. 面向临床研究的验证设计。

## 18. 验收指标

首次配置完成后，应满足以下产品目标：

* 用户或家属能够在 8 分钟内完成首次配置。
* 用户可以跳过任何任务。
* 用户不需要阅读长段文字。
* 每个设置都可在首页快速修改。
* 用户能看懂“为什么这样设置”。
* 用户可在 3 次点击内切换候选数量。
* 用户可在 2 次点击内关闭自动卡顿提醒。
* 所有 AI 候选均需要用户确认后播报。
* 用户可删除画像、词库、对话记录和个人物品。
* 家属可在不改动用户表达内容的前提下调整界面。

## 19. 比赛中的核心表述

“语桥并不使用 AI 给失语者贴上轻度、中度或重度标签，而是参考医学评估中的命名、理解、复述、阅读、书写和功能沟通维度，构建可解释的沟通能力画像。系统根据用户最容易表达的方式，动态调整候选数量、图片比例、句式支持、场景词库和沟通伙伴协作方式。”

“语桥关心的不是用户失语有多严重，而是用户在这一刻怎样最容易表达。”
