# 语桥首次配置 MVP V2

## 一、设计目标

首次配置的目标不是诊断失语症，而是帮助语桥选择更适合用户的表达方式。

核心原则：

* 不显示“轻度、中度、重度”等标签。
* 不输出医学诊断或康复结论。
* 用户随时可跳过、修改或恢复默认。
* 引导适配的结果只影响界面和辅助策略。
* 停顿辅助只保留“关闭 / 辅助”两种状态。
* 默认建议开启停顿辅助，但只有用户首次进入对话功能并授权麦克风后才真正生效。

## 二、首次启动流程

### 第 1 屏：隐私与使用说明

标题：

**使用前说明**

正文：

> 语桥用于日常沟通辅助，不用于医学诊断、治疗或紧急判断。
>
> 你可以自主选择是否使用拍照、语音、定位等功能；语桥只会在你主动使用相关功能时申请对应权限。
>
> 表达偏好将默认保存在本机。若启用云端识别或生成服务，语桥会在使用前说明相关内容的处理方式。
>
> 你可以在“我的—隐私与数据”中查看、修改或删除个人数据。

底部：

* 阅读隐私政策
* 同意并继续

实现要求：

* 隐私政策同意状态单独保存。
* 不在首次启动一次性申请相机、麦克风、定位权限。
* 相机、麦克风、定位均在用户主动进入对应功能时再申请。
* 若后续接入云端识别或生成，应在首次调用前单独提示。
* 偏好配置、引导结果第一版仅本地保存，不上传。

---

### 第 2 屏：谁在帮忙设置

标题：

**谁在帮你设置语桥？**

三个大按钮：

* 我自己设置
* 家人帮我设置
* 直接使用默认设置

默认设置：

```text
候选数量：2
显示方式：图文
当前状态：正常
停顿辅助：辅助
首页重点：今日快捷表达、拍照找词、我卡住了
```

说明：

* “家人帮我设置”只改变引导文案和操作节奏，不进入完整家属系统。
* “直接使用默认设置”可跳过后续全部步骤。
* 所有人都能在“我的—表达偏好”中重新配置。

---

### 第 3 屏：选择配置方式

标题：

**怎样开始更舒服？**

两个大卡片：

#### 快速设置

副标题：

**自己选择最舒服的表达方式**

适合：

* 用户或家属已经清楚偏好。
* 希望尽快进入首页。
* 不希望完成体验任务。

按钮：

**我自己选择**

#### 引导适配

副标题：

**通过几个小体验，为你推荐设置**

适合：

* 不确定选项数量、图片还是文字更适合。
* 希望语桥帮助推荐界面。
* 不想自己理解复杂设置。

辅助说明：

> 这不是医学测试，只用于推荐更适合你的表达界面。

按钮：

**让我试试看**

## 三、快速设置流程

共四项，每项一屏，大按钮、可试听、可返回。

### 1. 我最常需要什么帮助

标题：

**你最希望语桥帮什么？**

单选：

* 找不到想说的词
* 不容易组织成一句话
* 不容易听懂别人说的话
* 想快速表达日常需要

系统作用：

| 选择       | 首页重点入口 |
| -------- | ------ |
| 找不到词     | 拍照找词   |
| 不容易组织成句  | 我想说    |
| 不容易听懂    | 帮我理解   |
| 快速表达日常需要 | 今日快捷表达 |

首页不完全动态排序，只将对应入口作为“今日重点”展示。

---

### 2. 一页显示多少内容更舒服

标题：

**你觉得哪一种更容易选？**

三个大按钮：

* 少一些：2 个候选
* 标准：4 个候选
* 多一些：6 个候选

用户界面不显示“候选数量”专业词，统一使用：

* 少
* 标准
* 多

---

### 3. 你更喜欢怎么查看

标题：

**你更喜欢哪种显示方式？**

三个大按钮：

* 大文字
* 图文一起
* 图片优先

对应：

```text
大文字 → text
图文一起 → mixed
图片优先 → imageFirst
```

---

### 4. 今天状态怎么样

标题：

**今天想用得轻松一点吗？**

三个大按钮：

* 正常
* 有点累
* 很累

说明：

* 这是临时状态，不代表长期能力。
* 每天可在首页或“我的”中快速修改。
* “很累”时自动减少候选、弱化动画、优先显示常用表达。

完成后进入“设置结果页”。

## 四、引导适配流程

总时长控制在 90 秒以内。

不显示“测试”“得分”“正确”“错误”等词语，只使用“试试看”“哪种更舒服”。

### 体验 1：找词方式

标题：

**看到这个时，哪种帮助最适合你？**

展示一张真实物品图片，例如杯子。

选项：

* 我能直接说出来
* 我想先看到图片
* 我想先看到文字
* 我想听提示

系统作用：

* 推荐图片、文字或图文显示方式。
* 若多次选择图片帮助，则提高“拍照找词”和个人物品入口的优先级。

---

### 体验 2：表达一句话

标题：

**你想喝水时，哪种表达方式最容易？**

三个选项：

* 我想喝水
* 我想 + 喝水
* 图片：我 + 喝水

系统作用：

| 偏好   | 推荐表达方式   |
| ---- | -------- |
| 完整短句 | 高频完整句优先  |
| 分步组合 | 动态槽位补词优先 |
| 图文组合 | 图片辅助表达优先 |

---

### 体验 3：听懂对方

标题：

**听一听，选出最接近的意思**

播放短句：

> 请帮我拿水。

显示两到三张图片：

* 拿水
* 开门
* 坐下

可选操作：

* 再听一遍
* 我想看文字
* 我想看图片
* 跳过

系统作用：

* 若用户多次选择“再听一遍”或“看文字”，则在“帮我理解”中优先提供原句、关键词和重复播放。
* 不把结果解释为听理解障碍，不输出任何医学结论。

---

### 体验 4：选项密度

标题：

**哪一页最容易选择？**

用同一组常用需求分别展示：

* 2 个大卡片
* 4 个标准卡片
* 6 个紧凑卡片

让用户找到“喝水”。

记录：

* 用户主观偏好。
* 是否一次点中。
* 是否返回重新选择。
* 是否选择“我想要更少”或“我想要更多”。

系统规则：

```text
用户主动选择优先级最高。
体验结果仅作为推荐依据。
若用户主观偏好与体验结果不同，采用用户主动选择。
```

## 五、配置结果页

* 选项数量：标准

按钮：

* 开始使用
* 我想调整
* 使用默认设置

## 六、停顿辅助设计

停顿辅助不放入首次配置流程。

默认策略：

```text
profile.stuckAssistEnabled = true
```

但实际启用条件：

```text
用户首次进入对话功能
→ 展示说明
→ 用户允许麦克风权限
→ 停顿辅助生效
```

首次进入对话页提示：

> 开启后，语桥会在你说话停顿较久时尝试提供补词建议。
> 它可能猜错，所有建议都需要你确认后才会播报。

两个按钮：

* 开启辅助
* 暂不开启

设置页只保留：

* 停顿辅助：关闭 / 辅助

交互要求：

* 开启辅助时，对话页始终显示明显状态：“语桥正在辅助”。
* 提供一键暂停按钮。
* 候选永远需要用户确认。
* 连续多次被拒绝后，降低本次会话的触发频率。
* 用户关闭后，不自动重新开启。

---

## 七、首页结构

首页固定六个区域，不做频繁动态重排：

1. 今日快捷表达
2. 今日重点帮助
3. 拍照找词
4. 我卡住了
5. 帮我理解
6. 我的常用语

只允许“今日重点帮助”随 `primaryNeed` 改变：

| primaryNeed      | 今日重点帮助 |
| ---------------- | ------ |
| wordFinding      | 拍照找词   |
| sentenceBuilding | 我想说    |
| comprehension    | 帮我理解   |
| quickNeeds       | 今日快捷表达 |

图片优先时：

* “拍照找词”视觉更突出。
* “我的物品 / 人物 / 地点”入口放入“我的常用语”区域上方。
* 不改变其他核心入口位置。

---

## 八、疲劳状态覆盖规则

长期偏好与当天状态分开保存。

```text
长期偏好：preferredCandidateCount
当天状态：fatigueLevel
实际显示：effectiveCandidateCount
```

规则：

| 长期偏好 | 今天状态 | 实际显示 |
| ---- | ---- | ---- |
| 2    | 任意   | 2    |
| 4    | 正常   | 4    |
| 4    | 很累   | 2    |
| 6    | 正常   | 6    |
| 6    | 有点累  | 4    |
| 6    | 很累   | 2    |

疲劳模式附加效果：

| 状态  | 界面策略              |
| --- | ----------------- |
| 正常  | 完整功能              |
| 有点累 | 优先常用表达，减少一层跳转     |
| 很累  | 2 候选、大卡片、少动画、短句优先 |

## 九、CommunicationProfile MVP 数据结构

```dart
enum SetupBy { self, caregiver, defaultMode }

enum SetupMethod { quick, guided }

enum PrimaryNeed {
  wordFinding,
  sentenceBuilding,
  comprehension,
  quickNeeds,
}

enum DisplayMode { text, mixed, imageFirst }

enum FatigueLevel { normal, tired, veryTired }

class CommunicationProfile {
  final bool completedOnboarding;
  final bool privacyAccepted;
  final String privacyPolicyVersion;
  final DateTime privacyAcceptedAt;

  final SetupBy setupBy;
  final SetupMethod? setupMethod;

  final PrimaryNeed primaryNeed;
  final int preferredCandidateCount; // 2 / 4 / 6
  final DisplayMode displayMode;
  final FatigueLevel fatigueLevel;

  final bool stuckAssistEnabled;

  final DateTime updatedAt;

  const CommunicationProfile({
    required this.completedOnboarding,
    required this.privacyAccepted,
    required this.privacyPolicyVersion,
    required this.privacyAcceptedAt,
    required this.setupBy,
    required this.setupMethod,
    required this.primaryNeed,
    required this.preferredCandidateCount,
    required this.displayMode,
    required this.fatigueLevel,
    required this.stuckAssistEnabled,
    required this.updatedAt,
  });
}
```

第一版不加入：

* 医学量表名称和分数。
* 失语类型。
* 轻中重等级。
* 治疗师身份和账号体系。
* 病史、诊断、药物信息。
* 自动推断康复进展。
* 复杂画像分数。

## 十、现有功能接入规则

### 补词

```dart
final count = effectiveCandidateCount(profile);
```

AI 请求数量：

```text
请求数量 = 实际显示数量 + 2
展示数量 = 实际显示数量
```

多出的候选用于过滤：

* 重复词。
* 无关词。
* 过长句子。
* 不安全或不确定表达。

---

### 拍照找词

读取：

* `effectiveCandidateCount`
* `displayMode`
* `primaryNeed`

规则：

* 图片优先：拍照结果优先展示真实照片、物品名称和少量表达建议。
* 找词优先：拍照后直接进入“它是什么 / 我想怎么说”。
* 很累：只展示 2 个高置信候选和“都不是”。

---

### 对话辅助

读取：

* `stuckAssistEnabled`
* `effectiveCandidateCount`
* `displayMode`

规则：

* 关闭：只保留手动“我卡住了”。
* 辅助：根据当前停顿检测逻辑提供候选。
* 所有候选必须点击确认。
* 很累：候选只显示 2 个，优先常用词和短句。

## 十一、设置页入口

“我的”页面新增：

**表达偏好**

可修改：

* 今日状态
* 选项数量：少 / 标准 / 多
* 显示方式：大文字 / 图文 / 图片优先
* 停顿辅助：关闭 / 辅助
* 重新进行引导适配
* 恢复默认设置

“隐私与数据”页面独立：

* 隐私政策
* 权限管理
* 云端服务说明
* 删除表达偏好
* 删除个人词库
* 删除对话记录
* 删除拍照记录

## 十二、第一版验收标准

1. 用户可在 3 分钟内完成快速设置。
2. 用户可在 5 分钟内完成引导适配。
3. 用户可跳过任意体验任务。
4. 用户可在 3 次点击内修改选项数量。
5. 用户可在 2 次点击内关闭停顿辅助。
6. 候选数量会真实影响补词、拍照建议、对话建议。
7. 很累状态会真实减少候选并简化界面。
8. 首页核心入口位置固定，不因画像频繁变化。
9. 全部 AI 建议必须确认后播报。
10. 不展示医学诊断、等级、测试分数或失语类型。
