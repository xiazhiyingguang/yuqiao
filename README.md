# 语桥 Flutter MVP

语桥是一个面向轻度失语症用户的辅助表达 App。当前版本使用 Flutter 实现，优先支持 Android 真机调试。核心闭环是：

```text
补词 / 对话上下文 / 拍照识物
-> Qwen API 推荐候选词或整理成句
-> 用户确认
-> 系统 TTS 播报
```

## 当前功能

- 星星主界面：四个悬浮球进入 `补词`、`拍照`、`对话`、`词库`。
- 补词流程：先选择寻找、请求、感受或描述任务，再通过动态槽位逐步构造表达；Qwen 只生成当前槽位候选，本地词库作为失败兜底。
- AI 成句：根据零散关键词调用 Qwen 生成 2-3 条候选句。
- 确认播报：用户二次确认后才调用系统 TTS。
- 拍照识物：调用相机或相册，使用 Qwen 视觉模型返回多个可能物品。
- 多识别结果确认：用户先确认“这是什么”，再选择相关表达。
- 我的物品：可把识别结果保存为个人物品，记录参考照片、名称、类型、外观、备注和常用表达，并在后续拍照时辅助匹配。
- 对话模式：用户主动开启后，将 16kHz 单声道 PCM 音频流发送到讯飞实时语音转写大模型，使用角色盲分记录“说话者 1/2”上下文并基于确定性结果补词。
- 对话特殊词汇：Qwen 在后台从确定性转写中提取人名、具体地点、机构和个性化称呼；点击粉色语音球可在转录弹窗中查看高亮词，并由用户确认后保存到本机。
- 对话理解辅助：非用户说话者的最终转录提供“帮我理解”入口；Qwen 将选中句子拆成人物、时间、地点、动作和顺序等信息，显示简单解释与重点，并支持慢速 TTS 朗读或播报“请说简单一点”。
- 理解安全校验：原句始终保留，每个拆解项必须引用原句证据；代词无法确定时显示不确定性，数字或否定关系校验失败时不展示可疑解释。
- 半自动卡顿辅助：本地规则检测疑似表达卡顿，Qwen 只负责基于已确认上下文整理一条候选句；手机轻震并显示候选，用户点击确认后才会 TTS 播报。
- 对话稳定性：讯飞转写按片段 ID 合并临时结果与最终结果；断线后按 1、2、4 秒自动重连，连续失败后提供手动重新连接。
- 对话反馈闭环：选择候选会记录偏好的表达类型；“都不是，换一组”会排除当前候选并重新请求 Qwen，反馈仅保存在本机。
- 对话上下文压缩：界面保留最近确定转录，较早内容压缩为仅由原文组成的本地摘要，避免长对话持续增加延迟和费用。
- 语义完整性校验：AI 成句使用结构化的“证据 / 合理假设”输出并进行二次校验，避免不完整句和把推测写成确定事实。
- 阿里云 Paraformer 服务继续保留，供词库搜索框等普通短语音输入使用，不与对话模式的讯飞服务混用。
- 日常词库：维护人物、饮食、地点、活动、物品、感受、常用句等生活词汇。
- 本地保存：最近表达、常用语、日常词库使用 `shared_preferences` 保存。
- 地点推荐：前台定位、本地点簇、地点历史高频词和地点类型场景词共同参与候选排序。
- 地点记忆：支持确认或修改自动建议、管理地点名称/类型、查看和删除地点常用词。
- Android 真机运行：已在华为 HBN AL00 / Android 12 上完成 debug APK 安装。

## 关于 `liquid_glass_renderer`

已经验证并放弃把 `liquid_glass_renderer 0.2.0-dev.4` 用作主界面方案。

原因：

- 该库依赖 Fragment Shader 才能实现真正的光学折射。
- 当前华为 HBN AL00 运行时走 Skia 路径，`ImageFilter.isShaderFilterSupported` 为 false。
- 库会自动降级到 `FakeGlass`，本质也是 `BackdropFilter + blur + 手动画高光`，和当前自定义方案没有本质差异。
- 构建时会出现 SkSL/Impeller shader warning，容易干扰判断。

当前结论：

- 项目依赖中已经删除 `liquid_glass_renderer`。
- 测试页 `liquid_glass_test_page.dart` 已删除。
- 不再保留主界面 `Glass` debug 入口。
- 后续如果要重新尝试真实液态玻璃，需要等库稳定、Flutter Impeller 对当前设备兼容改善，或换用支持 shader filter 的设备。

## 对话模式：特殊词汇与卡顿辅助

### 数据流

```text
手机麦克风
-> 讯飞实时转写与说话者盲分
-> 确定性文本写入对话上下文
-> 本地卡顿规则检测
-> Qwen 根据上下文整理一条候选句
-> 轻震 + 用户确认
-> 暂停 ASR -> TTS 播报 -> 恢复 ASR
```

卡顿检测本身在手机本地完成，不把原始音频交给 Qwen。只有需要生成辅助表达时，才把已有的文字上下文、时间段、地点类型和个人词汇发送给 Qwen。地点只传类型，不发送精确经纬度。

### 用户说话者

- 讯飞返回 `speakerId`，页面默认把本次会话中第一个识别到的说话者视为用户。
- 点击粉色语音球，在“实时转录”弹窗中可以把“我的声音”切换为说话者 1、说话者 2 等。
- 只有当前 `speakerId` 等于用户选择的说话者时才执行卡顿检测，对方说话时不会触发用户卡顿辅助。
- 讯飞的说话者编号只代表当前会话角色，不能当作长期稳定的个人身份。

### 卡顿检测规则

当前第一版使用可解释的本地启发式规则，不使用大模型直接判断用户是否卡顿：

1. **实时片段停住**：讯飞仍返回非最终片段时启动 3.2 秒计时器；有新转写就取消旧计时器并重新计算，持续没有新内容才进入疑似卡顿。
2. **填充词**：检查“嗯、呃、那个、这个、就是、怎么说、我想想、等一下”等表达。
3. **未完成句式**：检查句子是否停在“我想、我要、能不能、帮我、给我、我要去、我需要、因为、但是、然后、还有”等结构上。
4. **重复表达**：检测连续字符、重复填充词，以及“我想、我要、帮我、给我、然后”等短语重复。
5. **最终片段等待**：最终识别文本只有命中上述异常规则时才等待 2.4 秒触发；普通完整句子后的沉默不会触发。
6. **对话继续则作废**：开始请求 Qwen 时记录对话版本；生成期间任何人继续说话，版本变化后会丢弃旧候选，避免弹出过期表达。
7. **并发与冷却**：生成或播报期间不再次触发；一次确认播报后进入 15 秒冷却时间。

当前规则仍可能误判，例如完整句子“这个很好”包含填充词列表中的“这个”。后续应改成多信号评分制，让停顿时长、未完成句式、重复、填充词和历史误判反馈共同达到阈值后再触发，而不是单个布尔条件命中就触发。

### 候选生成与播报保护

- Qwen 只能基于对话中已经明确出现的内容整理一句不超过 24 个汉字的候选，不能补充未经确认的人物、地点、症状或事实。
- 上下文不足时，模型应返回“请等一下，我还在想”一类安全沟通句。
- 候选生成后调用 `HapticFeedback.lightImpact()` 轻震，并显示“你可能想说”确认面板。
- 点击“不是”会关闭候选，不播报；点击“确认播报”才调用系统 TTS。
- TTS 前主动停止讯飞 ASR，播报完成后重新连接，避免手机把自己的播报声再次识别并形成回声循环。
- 确认播报会写入最近表达和地点使用记录，供后续推荐使用。

### 特殊词汇记忆

- 只处理讯飞返回的最终文本，按约 800ms 批量调用 Qwen，避免每个中间片段都发起请求。
- 提取类型包括 `person`、`place`、`organization` 和 `custom`。
- AI 提取结果只在当前对话中临时高亮，不会直接永久保存第三方对话信息。
- 用户点击高亮词，可以修正类型并确认保存；保存后的词汇使用 `SharedPreferences` 留在本机。
- 已确认词汇会提供给后续对话补词模型作为个人上下文，但不会无条件硬插入每一组候选。

### 真机测试

1. 使用包含 Qwen 和讯飞凭证的命令运行 App，进入对话模式并开启监听。
2. 点击粉色语音球，确认“我的声音”对应正确的说话者编号。
3. 说“我想……那个……”后停止约 3 秒，等待手机轻震和候选确认面板。
4. 点击“不是”，确认系统不会播报；再次触发后点击“确认播报”，确认讯飞监听暂停、TTS 播报、随后恢复监听。
5. 在模型生成期间继续说话，确认旧候选不会弹出。
6. 正常说完完整句子并保持安静，确认不会仅因普通轮次结束而触发。
7. 对话中说出明确人名或地点，稍等后点击粉色语音球，检查高亮词并尝试保存。
8. 确认“我的声音”选中正确说话者；在其他说话者的最终转录块下点击“帮我理解”，检查原句、分步拆解、简单解释和重点。
9. 点击“慢速朗读文字”，确认 ASR 暂停、TTS 结束后恢复；点击“请对方说简单一点”，确认只播报固定沟通句。
10. 断网或使用无效 Qwen Key 再请求理解，确认页面显示“暂时无法可靠解释”，不会编造解释或崩溃。

## 已完成记录

### 2026-06-21

- 重构独立补词功能，明确与对话模式分离；补词上下文只包含任务类型、用户记得的片段、已确认槽位和拒绝候选。
- 首屏改为“找人或东西、想做一件事、表达感受、描述一件事”四种任务，并支持键盘或阿里云 Paraformer 输入零散文字。
- 新增 `StuckExpressionSession` 动态状态机：不同任务使用不同槽位，“带我去”会进入地点槽位，情绪表达不会强制选择身体部位。
- Qwen 补词改为结构化候选，返回 `text + slot + semanticGroup`；本地再次校验槽位、内容长度和词语类型。
- 每组优先展示四种不同语义方向，避免同义词占满候选；已展示候选会进入排除集合。
- “换一组”第一次优先使用未展示模型候选，第二次要求 Qwen 改变语义方向，连续三次仍不合适时打开澄清面板。
- 澄清面板支持文字补充、语音补充、返回修改、重选任务和继续生成，不再显示“没有更多候选”的死路提示。
- 页面顶部新增可编辑表达轨迹；点击任一已选项会回到该槽位，并清除依赖它的后续选择。
- 为四类任务增加最低成句条件；达到条件后由用户选择“现在整理成句”或继续补充，不再固定第三步后自动跳转。
- 个人词库先按当前槽位分类过滤；地点和历史词只有通过语义检查后才能小幅排序或补位，不能越过 Qwen 与当前步骤候选。
- 补词 Qwen 请求增加取消令牌；返回、修改、切换任务和退出页面都会使旧请求失效，避免迟到结果覆盖新页面。
- 新增纯 Dart 状态验证脚本和状态机测试，覆盖动态切槽、最低成句条件、修改清理和槽位过滤。
- 修复补词文字/语音补充弹窗关闭时的红屏：原实现会在底部弹窗关闭动画结束前提前销毁 `TextEditingController`，现改为由独立弹窗 State 持有并在 Widget 真正卸载后销毁；同时返回时先取消模型请求和语音输入。
- 补词生成完整句子后，候选句页和确认播报页的左上角返回键、安卓系统返回键都会一次退出完整补词流程，不再逐层返回多个页面；其他功能入口保持原有返回逻辑。
- 修复补词第一轮容易显示无关模板的问题：外层 Qwen 等待窗口从 5 秒调整为 12 秒，避免首次 TLS 连接或模型冷启动被提前取消；所有 Qwen 请求复用 HTTP Client，并使用可中止请求保留退出取消能力。
- 补词提示词把用户片段和已确认槽位提升为最高优先级；有用户上下文时不再把本地模板发送给模型，防止模板带偏预测。

## 踩坑记录

### `flutter clean` 删除 `build` 失败

现象：

```text
Failed to remove build. A program may still be using a file in the directory.
```

本次定位到被占用文件：

```text
D:\code\yuqiao\build\app\outputs\flutter-apk\app-debug.apk
```

常见原因：

- `adb.exe` 正在安装或读取旧 APK。
- Android Studio、资源管理器、杀毒软件占用 APK。
- 之前的 Flutter/Gradle/Dart 进程未退出。

处理方式：

```powershell
taskkill /F /IM adb.exe
taskkill /F /IM java.exe
taskkill /F /IM dart.exe
Remove-Item -LiteralPath .\build\app\outputs\flutter-apk\app-debug.apk -Force
Remove-Item -LiteralPath .\build -Recurse -Force
```

如果仍然被占用，可以用 Windows 资源监视器：

```text
Win + R -> resmon -> CPU -> 关联的句柄 -> 搜索 app-debug.apk
```

结束占用该文件的进程后再删除。

## 如何确认 Qwen 是否真的调用

为了避免候选突然变化和持续显示加载动画，当前补词流程不会长期展示“Qwen 推荐成功”提示。模型结果准备好后再整体显示候选。

独立补词会调用 `recommendGuidedOptions`，请求中包含当前 `slot`、已确认内容、排除候选和换组层级。模型必须返回结构化 JSON；缺少 `QWEN_API_KEY`、超时或格式不正确时，页面等待到 5 秒后使用经过当前槽位过滤的本地候选。对话卡顿辅助生成失败时会输出 `[Conversation stuck assist]` 日志，不会伪造模型结果或自动播报。

## 独立补词状态机

- 补词不使用实时对话转录，也不会把地点类型当成当前意图。
- 表达会话记录 `intent`、`seedFragment`、各槽位选择、跳过槽位和已拒绝候选。
- 寻找任务确认对象后即可成句；请求任务必须有动作以及该动作需要的对象或地点；感受任务必须有具体感受；描述任务必须同时有主体和动作。
- 每个候选都有明确 `slot` 和 `semanticGroup`。模型即使在 JSON 中声称槽位正确，本地仍会检查时间、地点、身体部位、感受、动作等词形是否合理。
- 模型候选优先级高于本地词；地点和历史匹配最多提供小幅排序加分，并且上下文扩展词最多补充两项。
- 修改表达轨迹中的某项时，该项及后续依赖项会清除，状态机按新选择重新决定下一槽位。

验证状态机：

```powershell
E:\flutter\bin\cache\dart-sdk\bin\dart.exe run tool\verify_stuck_flow.dart
```

真机建议依次测试：

1. 选择“想做一件事 → 带我去”，确认下一步询问地点；改选“陪陪我”，确认不再要求物品或地点。
2. 选择“表达感受 → 有点害怕”，确认下一步进入程度，而不是身体部位。
3. 只确认寻找对象，检查“现在整理成句”和继续补充能否同时使用。
4. 点击表达轨迹中的第一项，确认后续选择被清除并重新推荐。
5. 连续点击三次“换一组”，确认出现澄清面板而不是“没有更多候选”。
6. 使用文字和语音补充一个未出现在候选中的词。

## 对话上下文与语义校验

- 讯飞临时转写不会直接追加成新句，而是按 `segmentId + speakerId` 更新同一个片段；最终结果到达后冻结该片段，减少末尾文字延迟造成的重复和错位。
- 用户继续说话、退出页面或关闭对话时，会取消进行中的对话补词、卡顿辅助和特殊词提取请求；Qwen 的 HTTP 客户端会随取消令牌关闭。
- Qwen 遇到超时、临时网络错误、`429` 或常见 `5xx` 时最多重试一次；讯飞 WebSocket 断开不会直接关闭对话模式。
- 对话超过 12 条确定片段时，只保留最近 8 条；更早内容从原始转写中提取最多 6 个短片段形成本地摘要，不调用模型，也不会产生新事实。
- AI 成句要求 Qwen 为每条句子返回 `evidence`、`assumptions` 和 `complete`。代码会检查 evidence 是否真的来自输入，并过滤语法未完成的句子。
- 新人名、地点、数字或症状不是自动判错。它们可以作为合理假设出现在询问、请求、建议或可能性表达中，例如“王医生在吗？”；只有把没有依据的假设写成已确认事实时才会被过滤并请求模型重写。
- 候选反馈按“表达槽位 + 最近转录片段”区分语境，避免一次拒绝影响所有对话；最多保留最近 60 条本地反馈。

对话模式建议测试：

1. 连续说两句并观察同一临时片段是否被更新，而不是重复追加；最终文字应自动滚动到最下方。
2. 点击“我卡住了”，选择一个候选；再次在相似语境触发时，观察所选表达类型是否优先。
3. 点击“都不是，换一组”，确认新一组不重复刚才四项。
4. 请求生成期间继续说话或关闭对话，确认旧候选不会在稍后突然弹出。
5. 临时断网后恢复，确认系统自动重连；三次失败后点击“重新连接”。
6. 测试“王医生在吗？”这类合理询问可以出现，而无上下文的确定诊断、用药数量等事实不会直接成为候选句。

## 地点词汇推荐（第一版）

- “我的”页面提供“开启地点词汇推荐”开关，默认关闭。
- 仅在开关开启且 App 前台使用时请求“使用期间位置权限”。
- 使用 `geolocator` 获取位置，3 分钟内复用缓存，不使用后台定位、地图、POI 或逆地理编码。
- 经纬度、地点簇和地点词频仅通过 `SharedPreferences` 保存在本机，不会传给 Qwen 或其他服务。
- 定位精度差于 120 米时不会创建地点；刷新定位本身也不会创建地点。
- 只有候选选择、表达确认、播报、收藏或词库保存等有效行为才会创建“常去地点 N”并累计词频。
- 同一地点默认按 120 米半径匹配；地点高频、近期、收藏及页面分类匹配词会在原候选中优先排序。
- “我的”页面可以随时清除全部地点簇和地点词频，不影响普通词库与表达记录。

真机测试顺序：

1. 打开“我的”，开启地点词汇推荐并允许“仅使用期间”定位。
2. 进入补词、对话、拍照或词库页面，选择或保存几个词。
3. 返回相同位置再次进入对应页面，确认曾使用词在候选中优先出现。
4. 关闭定位服务或拒绝权限，确认原功能仍可正常使用。
5. 在“我的”中清除地点记录，确认地点数量归零且普通词库未被删除。
