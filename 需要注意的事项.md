# 需要注意的事项

## 1. 星星中心的圆形阴影

**文件：** `lib/star_home.dart`
**位置：** `_drawPuffyFacets` 方法，第 1149–1161 行
**严重程度：** 中

**问题描述：**
`centerGlowPaint` 在星星正中心画了一个圆形径向渐变光晕，但 alpha 过渡不平滑。从 stop 0.36（alpha 0.20）到 stop 1.0（alpha 0.0）的跨度太大，形成亮度突变边界，在华为手机上表现为可见的圆形阴影。

**相关代码：**
```dart
stops: const [0.0, 0.36, 1.0],
colors: [
  Colors.white.withValues(alpha: 0.40),
  const Color(0xFFFFF6D6).withValues(alpha: 0.20),
  Colors.white.withValues(alpha: 0.0),
],
```

**次要因素：** 底部椭圆地面阴影（第 943–953 行）的 `MaskFilter.blur` 半径 14px，模糊渗透范围约 42px，可透过星星内凹处被看到。

**根本原因：** 代码问题，华为 Mali GPU 的 alpha 混合精度和色彩管理差异会放大此视觉瑕疵。

## 2. `liquid_glass_renderer` 在华为手机上不可用，当前手动方案是正确的

**文件：** `lib/star_home.dart`
**位置：** `LiquidGlassSurface` 组件，第 579–668 行
**严重程度：** 信息（无需修改）

**背景：**
`liquid_glass_renderer 0.2.0-dev.4` 通过 Fragment Shader 实现真正的光学折射效果，但**在当前华为手机上无法使用**。

**不可用原因（三层阻断）：**

1. **库是预发布版**（0.2.0-dev.4）：shader 实现不完善，本身不稳定
2. **华为手机走 Skia 路径**：该库需要 Impeller 渲染引擎支持 Fragment Shader，但华为 HBN AL00（麒麟芯片 + Mali GPU）与 Impeller 兼容性差，实际运行时回退到 Skia，Skia 不支持自定义 Fragment Shader
3. **Mali GPU shader 兼容性差**：即使强制启用 Impeller，Mali GPU 的 Vulkan/OpenGL ES 驱动对复杂 shader 支持不完整，可能黑屏或闪退

**库的自动降级机制已验证（源码确认）：**
`LiquidGlassLayer` 的 build 方法（`lib/src/rendering/liquid_glass_layer.dart` 第 124 行）会检测 `ImageFilter.isShaderFilterSupported`，若为 false（Skia 路径）则自动降级为 `FakeGlass`。`FakeGlass` 的注释明确写道：
> "A widget that aims to provide a similar look to LiquidGlass, but **without the expensive shader**."
> "Some properties will not have any effect, such as `thickness` and `refractiveIndex`, since there is **no actual refraction happening**."

`FakeGlass` 的实现就是 `BackdropFilter` + `ImageFilter.blur` + 手动画高光——与当前 `LiquidGlassSurface` 的做法本质相同，不会有更好的效果。

**已验证的问题（见 README.md 第 37 行）：**
> 暂不使用 `liquid_glass_renderer 0.2.0-dev.4`：该版本 shader 在当前 Android/Skia 兼容路径下会产生 SkSL 编译警告，并可能导致真机闪退。

**当前方案评估：**
使用 `BackdropFilter` + `ImageFilter.blur` + 半透明白色渐变是当前设备上**唯一能稳定运行的液态玻璃近似方案**。虽然缺少光学折射，但模糊+渐变+高光+阴影的组合已经能传达液态玻璃的视觉方向。

**如果未来想用真正的液态玻璃：**
需要等 `liquid_glass_renderer` 发布稳定版 + Flutter Impeller 对 Mali GPU 兼容性改善，或换用支持 Impeller 的设备（如高通芯片 Android 手机或 iPhone）。

## 3. `thickness` 参数声明了但从未使用

**文件：** `lib/star_home.dart`
**位置：** `LiquidGlassSurface` 第 584 行
**严重程度：** 低

**问题描述：**
`LiquidGlassSurface` 声明了 `final double thickness` 参数（默认值 20），但在整个 `build` 方法中完全没有被引用。这是一个空壳参数，说明在实现时考虑过厚度概念但未落地。

## 4. `FeatureDetailPage` 未复用 `LiquidGlassSurface`

**文件：** `lib/star_home.dart`
**位置：** `FeatureDetailPage` 第 1389–1419 行
**严重程度：** 低

**问题描述：**
`FeatureDetailPage` 中的玻璃卡片手动用 `ClipRRect` + `BackdropFilter` + `Container` 实现，没有使用已有的 `LiquidGlassSurface` 组件。导致效果不一致（缺少顶部高光条、缺少双层阴影）且维护困难。

## 5. `main.dart` 中 `LiquidBackButton` 未复用 `LiquidGlassSurface`

**文件：** `lib/main.dart`
**位置：** `LiquidBackButton` 第 2309–2356 行
**严重程度：** 低

**问题描述：**
`LiquidBackButton` 手动实现了磨砂玻璃效果，alpha 值和渐变颜色与 `LiquidGlassSurface` 略有不同（如 opacity 0.78/0.44 vs 组件默认 0.67/0.45），未复用统一组件，导致视觉细节不一致。
